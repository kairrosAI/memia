How MEMIA BPOS works.

This guide summarizes the daily operating mode: install, launch a project, capture a signal, update memory, connect surfaces, use agents and keep control.

Install

The nominal path uses npm once the package is published. Before npm publication, GitHub and ZIP remain the distribution paths.

npx @kairros/memia bootstrap ./MEMIA --git --obsidian cd MEMIA memia status

Daily operating mode

The natural mode is conversational. The user asks an AI assistant to work inside MEMIA. Commands are used to verify, automate and integrate.

Launch a MEMIA project for this initiative. Create the structure, record decisions, prepare the handover and propose the next action. Then update MEMIA memory with the decisions we just made.

Launch a project

A MEMIA project should contain a goal, context, decisions, risks, resources and a handover. It should be resumable by another person or another AI session.

  • Create the project.
  • Capture useful context.
  • Identify initial decisions.
  • Add next actions.
  • Prepare the handover.

Captures

A capture is a signal not to lose. It can come from an idea, meeting, email, message, file or webhook. It should be qualified before it is routed.

memia new capture "Important signal" memia capture surface gmail "Email to qualify" memia capture surface whatsapp "Message to process"

Surfaces

Surfaces are where signals arrive: Gmail, WhatsApp, calendars, meetings, files, Drive, SharePoint, webhooks or business tools. A surface should not become unlimited permission. It reads, captures, routes and prepares under explicit rules.

memia surfaces init memia surfaces list memia surfaces whatsapp qr

Agents

MEMIA agents frame the work. Core agents apply the system. The board challenges decisions. Specialized agents cover business domains.

memia agents list memia agents list board memia agents show memia-project-steward

Security

MEMIA BPOS follows a local-first model. Secrets are not versioned. Sensitive external actions require explicit approval: sending, publishing, deleting, sharing, paying or signing.

Distribution

GitHub publishes the source. npm enables `npx`. ZIP supports standalone distribution. Homebrew can follow. The `memia.ai` website must serve only the static `site/` folder.
