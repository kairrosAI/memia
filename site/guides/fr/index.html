Mode de fonctionnement MEMIA BPOS.

Ce guide résume le fonctionnement quotidien : installer, lancer un projet, capturer un signal, mettre à jour la mémoire, brancher des surfaces, utiliser les agents et garder le contrôle.

Installation

Le chemin nominal passe par npm une fois le package publié. Avant publication npm, GitHub et ZIP restent les chemins de distribution.

npx @kairros/memia bootstrap ./MEMIA --git --obsidian cd MEMIA memia status

Fonctionnement quotidien

Le mode naturel est conversationnel. L’utilisateur parle à son assistant IA et lui demande de travailler dans MEMIA. Les commandes servent à vérifier, automatiser et intégrer.

Lance un projet MEMIA pour préparer ce sujet. Crée la structure, note les décisions, prépare le handover et propose la prochaine action. Mets ensuite à jour la mémoire MEMIA avec les décisions prises.

Lancer un projet

Un projet MEMIA doit contenir un objectif, un contexte, des décisions, des risques, des ressources et un handover. Il doit être reprenable par une autre personne ou une autre session IA.

  • Créer le projet.
  • Capturer le contexte utile.
  • Identifier les décisions initiales.
  • Ajouter les prochaines actions.
  • Préparer le handover.

Captures

Une capture est un signal à ne pas perdre. Elle peut venir d’une idée, d’une réunion, d’un email, d’un message, d’un fichier ou d’un webhook. Elle doit être qualifiée avant d’être routée.

memia new capture "Signal important" memia capture surface gmail "Email à qualifier" memia capture surface whatsapp "Message à traiter"

Surfaces

Les surfaces sont les endroits où arrivent les signaux : Gmail, WhatsApp, calendriers, réunions, fichiers, Drive, SharePoint, webhooks ou outils métier. Une surface ne doit pas devenir une permission illimitée. Elle lit, capture, route et prépare selon des règles explicites.

memia surfaces init memia surfaces list memia surfaces whatsapp qr

Agents

Les agents MEMIA cadrent le travail. Les agents cœur appliquent le système. Le board challenge les décisions. Les agents spécialisés couvrent les domaines métier.

memia agents list memia agents list board memia agents show memia-project-steward

Sécurité

MEMIA BPOS suit une logique local-first. Les secrets ne sont pas versionnés. Les actions externes sensibles demandent une validation explicite : envoi, publication, suppression, partage, paiement ou signature.

Distribution

Le dépôt GitHub publie les sources. npm rend `npx` disponible. ZIP permet une distribution autonome. Homebrew peut suivre. Le site `memia.ai` doit publier uniquement le dossier statique `site/`.
