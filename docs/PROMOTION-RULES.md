# Règles de promotion

La promotion transforme un contenu temporaire en mémoire durable.

## Ce qui peut être promu

- une capture qualifiée ;
- une synthèse issue d'un workspace ;
- une décision ;
- une méthode ;
- un livrable final ;
- un pointeur vers un fichier externe.

## Ce qui ne doit pas être promu tel quel

- un dump brut ;
- un export non relu ;
- un cache ;
- un fichier sensible ;
- une sortie intermédiaire non validée ;
- un document lourd sans raison.

## Processus

1. Lire la source.
2. Identifier l'intention utile.
3. Supprimer le bruit.
4. Réécrire en langage clair.
5. Choisir une destination IPCRWA.
6. Ajouter une trace dans `HANDOVER.md`, `DECISIONS.md`, un registre ou une fiche ressource.
7. Garder un pointeur vers les fichiers externes quand nécessaire.

## Décision minimale

Toute promotion doit répondre à quatre questions :

```text
Pourquoi ce contenu devient-il durable ?
Où vit-il ?
Quelle est la prochaine action ?
Comment revenir à la source ?
```
