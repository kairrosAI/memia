Page 55

Connecting a surface

Connecting a surface starts with a local configuration. MEMIA does not ask for secrets in documentation, does not store tokens in Git and does not mix read activation with external action.

memia surfaces init
memia surfaces list
memia surfaces status

For Gmail, enable a limited read mode. The actual connector may use OAuth, IMAP or controlled exports depending on the environment.

memia surfaces connect gmail --mode oauth --window 7 --limit 50

Checkpoint: after activation, run memia surfaces status and confirm that external actions still require human approval.

Page 56

WhatsApp QR

WhatsApp can be connected through a local bridge or the official API. In local bridge mode, MEMIA BPOS asks the bridge for a QR, writes the result into _memia/runtime/surfaces/whatsapp/, then the user scans the QR in WhatsApp.

memia surfaces connect whatsapp --mode local-bridge --bridge-url http://127.0.0.1:BRIDGE_PORT
memia surfaces whatsapp qr
memia surfaces status

The bridge must expose at least one compatible endpoint: /session/qr, /qr, /api/whatsapp/qr or /whatsapp/qr. MEMIA accepts a text payload or a data:image/... image.

Checkpoint: MEMIA must not send a WhatsApp message without explicit approval of the exact outgoing message.

Page 57

Surface contract

A surface declares its id, inbox, modes, expected environment variables, useful captures and action policy. The local configuration lives in connectors/surfaces.local.json, ignored by Git.

The minimal connector contract is limited read, local capture, IPCRWA routing, evidence, rollback and L5 approval for external actions.

Checkpoint: before production, ask MEMIA Security Officer to review scopes, secrets, logs and shutdown.

Page 58

Governed loops

A governed loop turns a surface into a controlled routine. It declares intent, trigger, surface, read scope, allowed actions, blocked actions, L0-L5 level, owner, evidence, rollback and review cadence.

cp templates/GOVERNED-LOOP.md "03 - Ressources & Assets/MEMIA/gmail-weekly-loop.md"

L0 to L4 cover observation, local capture, draft, internal update and approved local scheduled routine. L5 covers sending, deleting, archiving, sharing, paying, signing, publishing, inviting or changing permissions.

Checkpoint: an L5 loop is blocked by default.

Page 59

Loop states

A loop can be draft, enabled_read, enabled_draft, blocked_approval, active, paused or retired.

These states make operations readable. Another assistant must understand whether the loop is only described, reading, drafting, active, paused or retired.

Checkpoint: every active loop needs an owner, cadence and rollback.

Page 60

MEMIA BPOS Board

The MEMIA BPOS board is used when a decision needs several angles. It includes MEMIA Board Chair, MEMIA Product Strategist, MEMIA Architecture Lead, MEMIA Security Officer, MEMIA Operations Lead and MEMIA Data Privacy Steward.

memia agents list board
memia agents show memia-board-chair
memia agents show memia-security-officer

Typical request: “Run the MEMIA board on this surface activation and give me the recommendation, risks, L0-L5 level and trace to write.”

Checkpoint: the board recommends; the user approves sensitive actions.

Page 61

Security agent

MEMIA Security Officer reviews secrets, permissions, scopes, logs, sensitive data, external actions and shutdown. It is used before enabling a connector, bridge, webhook, scheduled routine or public export.

Its output should classify the decision as go, go_limited or blocked, with reasons and mitigation actions.

Checkpoint: no secret should appear in a distributed file, public HTML, commit or open conversation.

Page 62

Conversational use with surfaces

The user can speak naturally: “Connect WhatsApp in read mode through the local bridge”, “Run this week’s Gmail signal review”, “Update the brain”, “Prepare the governed loop”, “Run the board before activation”.

The assistant translates these phrases into CLI or MCP calls, explains what it will create, writes captures or governance files, then asks for approval whenever an action leaves the local brain.

Checkpoint: the product experience is conversational; commands are the technical support layer.
